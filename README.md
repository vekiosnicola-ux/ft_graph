# 🌊 ft_graph: Swimming Through 42 Together

Welcome to **ft_graph**, a living, breathing map of my journey through the 42 Piscine. 🎯

In the spirit of 42, this isn't just a personal notebook—it's an **Open Source Knowledge Base** built for the community. The Piscine is designed to be a high-pressure, communal "swimming" experience where we learn more from each other than from any manual. This vault is my contribution to that collective effort.

This project is built on the belief that **knowledge grows when it's shared**. If you're a fellow Pisciner (or a future one), feel free to:

- **Contribute**: Found a better way to explain a nested pointer or a tricky Norm rule? Submit a PR!
- **Share**: Use these diagrams and notes to help your peers during evaluations.

## 🧠 The Vault Philosophy
The 42 Piscine can feel chaotic. This vault is organized using **Cognitive Science principles** to help you stay afloat:
- **Visual Memory**: Every conceptual area is color-coded to reduce cognitive load (Blue for logic, Pink for active recall).
- **Semantical Hardlinking**: There are no "orphans" here. Every concept is connected to the modules where it applies.
- **AI-Augmented**: If you use Obsidian's "Smart Connections," you can literally "talk" to the vault to find dependencies across 370+ notes.

## 🎨 Knowledge Legend (At a Glance)
- 🟠 **Flashcards**: Active recall for when you're prepping for the Friday exams.
- 🔵 **C-Modules**: The core labs, broken down by function and mission.
- 🟡 **Concepts**: The "Why" behind the "How" (Pointers, Memory, Data Structures).
- 🔴 **Exams**: Strategies and simulations to survive the final push.
